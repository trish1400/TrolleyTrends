This relates to the Trolley Trends site located at https://www.trolleytrends.com

Personal data is not collected

The point of this site is to provide client-side visualisation of your loyalty card data. Client-side means the data is not submitted to a server, the processsing and visualisations are created using javascript in your browser, without the data leaving your device. No personal data is collected. You can enable Airplane mode (or similar) before submitting your json file.

If you opt to contribute your data, then the data is anonymised, client-side in your browser, before it is submitted. This data is anonymised, not just pseudo-anonymised. See the bottom of this modal for more details on this.

As no personal data is collected, all of this is outside of the scope of GPDR and similar legislation, however, I think transparency is important hence all these words.

Processors

The site is hosted with Cloudflare (about 20% of web goes through Cloudflare in some form).

If you submit your anonymised data, it goes via an Cloudflare Worker into Azure Blob Storage and then a Azure Function App puts it into an Azure SQL database.

Cookies

I don't set any first party cookies.

Cloudflare uses some strictly necessary cookies. You can read Cloudflare's privacy policy and Cloudflare's cookie policy.

Analytics

The only analytics are Cloudflare web analytics. I get twitchy about Google Analytics (what does 'consent mode' really do?) and anything like Hotjar in this context would be completely against the principles of retaining your data clientside. Cloudflare Web Analytics does not use any client-side state, such as cookies or localStorage, to collect usage metrics. It also doesn't 'fingerprint' individuals via their IP address, User Agent string, or any other data for the purpose of displaying analytics.

I've also not used any CDNs for third-party packages to avoid any potential for tracking there.

What about IP addresses, aren't they personal data?

No, it depends on the context, if you are an ISP, then maybe. For most services using IP addresses, it makes no sense, at worst they could be a key to link psuedo-anonyomised datasets, but if you have no other personal data, the idea that IP addresses alone are personal data makes no sense. The EU General Court ruled so in April 2023 SRB v EDPS.

Where possible, logs are kept to a minimum, not least because I'm not paying to store logs.
